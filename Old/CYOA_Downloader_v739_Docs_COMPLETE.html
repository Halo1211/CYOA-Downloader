CYOA Downloader v7.3.9

Desktop tool for downloading and packaging Interactive CYOAs for offline playback.
Python · GUI + CLI · v7.3.9 patched · FlareSolverr Cloudflare mode · GUI + CLI parity

Overview

CYOA Downloader downloads Interactive CYOA projects from the web and packages them for offline use. It supports multiple viewer engines (ICC Plus, ICC Remix, cyoap_vue, custom React/Vue), handles image/audio embedding, font downloading, and can produce complete offline-playable folders with injected viewer packages.

Supported sources:

  • Direct CYOA URLs (neocities, netlify, vercel, GitHub Pages, etc.)
  • cyoa.cafe links (automatic iframe resolution)
  • archive.org CYOA catalogs (auto-redirect to original sites)
  • Any URL serving an ICC-compatible project.json

Output formats:

  • Website folder — full site download with all assets, ready to open in browser
  • Embedded JSON — single .json file with base64-encoded images
  • ZIP archive — project.json + images/ folder
  • Both — embedded JSON + ZIP simultaneously
  • Offline viewer folder — injected into a real ICC viewer for direct playback

Quick Start

GUI — Download a CYOA in 3 steps

  1. Launch the app. Paste the CYOA URL into the URL bar.
  2. Click Add to Queue (or press Enter). The mode auto-detects.
  3. Click ▶ Start. Output appears in the configured output folder.

This auto-detects the best mode, downloads the project, processes images, and saves everything to ./out.

Download Modes Detail

🌐 Website Mode

Downloads the entire website into a local folder. This is the most comprehensive mode — it crawls HTML, CSS, JS, images, fonts, and other assets, preserving the original directory structure.

Uses WebsiteDownloader class (30 methods) which handles:

  • Recursive HTML crawling with CSS/JS asset extraction
  • Concurrent downloads via ThreadPoolExecutor
  • URL rewriting (absolute → relative) for offline playback
  • Integrity validation (checks all local references exist)
  • Project JSON injection into the downloaded viewer
  • BFS deep scan for dynamically-loaded assets

Output: a folder like MyProject/ containing index.html + all assets. Open index.html in a browser to play.

📦 Embed Mode

Downloads the project JSON and embeds all images as base64 data URIs directly in the JSON. Output is a single .json file that can be loaded into any ICC viewer.

Warning: Embedded files can be very large (50-500+ MB) for image-heavy CYOAs because base64 encoding inflates size by ~33%. Consider ZIP mode for large projects.

🗜 ZIP Mode

Downloads project JSON + all images into a temporary folder, then zips it. The JSON contains local relative paths (images/foo.png) instead of URLs. Audio files go into an audio/ subfolder.

📦+🗜 Both Mode

Produces both an embedded JSON and a ZIP archive simultaneously. Useful when you want the convenience of a single file AND the smaller zip for archival.

cyoap_vue Mode

Specialized mode for CYOAs built with the cyoap_vue engine. Downloads data.json + pass.json, images from images/, and media assets from Firebase or other backends. Output is a folder matching the cyoap_vue structure.

Download Pipeline

URL input │ ├─→ Phase 1: URL Resolution │ cyoa.cafe → iframe → real URL │ archive.org → original site │ Google Sheets → CSV export │ ├─→ Phase 2: Project Detection (get_project_source) │ 60+ candidate URLs (HEAD check, parallel) │ HTML <script> scanning │ Embedded JS extraction (balanced brace) │ ZIP/archive payload extraction │ [AI Phase 5: Claude fallback] │ ├─→ Phase 3: Asset Processing (process_images) │ JSON deep walker (IMAGE_FIELDS, AUDIO_FIELDS) │ Regex scanner (15 patterns) │ Parallel download + embed/save │ YouTube audio (yt-dlp) │ Font scanning + download │ ├─→ Phase 4: Output & Offline Viewer │ Save JSON (embed / zip / both) │ Inject into offline viewer │ CYOA Manager registration │ Metadata save │ └─→ Phase 5: Deep Scan (website mode only) BFS iterative scan (max 6 rounds) Connection-pooled single-pass GET Manifest probing (Vite/Webpack/CRA) [AI round: Claude JS analysis]

Phase 1: URL Resolution

Normalizes the input URL. Special handling for:

  • cyoa.cafe — resolves the iframe chain to the actual CYOA URL
  • archive.org — extracts original site URL from CYOA Manager catalog entries
  • Google Sheets — converts share URL to CSV export URL
  • Auto-detect — probes the URL with HEAD checks to determine if it's a website, embed, or cyoap_vue site

Phase 2: Project Detection

The get_project_source() function finds the project JSON. Strategy order:

  1. Phase 1 — Candidate HEAD check: 60+ candidate URLs built from URL patterns (/project.json, /app/project.json, Vite asset paths, etc.). Parallel HEAD checks find which exist.
  2. Phase 2 — HTML scan: Parse <script> tags for project data or JS bundle references.
  3. Phase 3 — Embedded JS extraction: Balanced-brace extraction from app bundles. Handles Vite/Webpack minified output.
  4. Phase 4 — Iframe resolution: If the page has iframes, recurse into them.
  5. Phase 5 — AI fallback: (requires API key) Ask Claude to analyze the HTML and locate the project URL.

Phase 3: Asset Processing

process_images() handles all asset types:

JSON Deep Walker (26 audio fields, 20+ image fields)

Walks the entire project JSON recursively. For each object, checks all known field names for asset URLs:

  • IMAGE_FIELDS: image, backgroundImage, rowBackgroundImage, objectImage, titleImage, etc.
  • AUDIO_FIELDS: audio, audioSrc, backgroundMusic, bgm, sfx, ambience, voiceover, clickSound, hoverSound, etc. (26 fields)
  • BGMLIST_FIELDS: bgmList, musicList, soundList, ambientList
  • HTML-in-JSON: Extracts <img src> from text/HTML fields
  • Markdown: Extracts ![alt](url) from text fields
  • CSS-in-JSON: Extracts url() from inline styles
  • Relative paths: Catches values that look like file paths even outside known fields

Regex Scanner (15 patterns)

After JSON walking, regex patterns catch anything missed:

  1. Double-quote string literals with asset extensions
  2. Single-quote string literals (critical for minified JS)
  3. CSS url() in all file types
  4. CSS image-set() responsive declarations
  5. HTML srcset attributes
  6. data-src, data-lazy, data-background, poster attributes
  7. <source src> elements
  8. <link rel="preload">
  9. SVG <image href> / xlink:href
  10. JSON manifest recursive walk (Vite/Webpack/CRA)
  11. Service Worker __precacheManifest
  12. Video extensions (.mp4, .webm, .mov)

Phase 4: Offline Viewer Injection

After downloading, the project can be injected into a registered offline viewer. Three strategies, tried in order:

  1. Strategy A (Template): ICC Remix viewers have {{ICC_PROJECT_DATA_SCRIPT}} markers in index.html. Replace with the project data.
  2. Strategy B (Marker): ICC Plus / New Viewer have /* Delete and replace this part */ comments in app.js. Inject project JSON at the marker using balanced-brace analysis.
  3. Strategy C (Interceptor): Fallback — inject a fetch/XHR interceptor into <head> that intercepts all requests for project.json and returns the embedded data.

A cheat overlay (gear button with point editor, requirement remover, select all) is injected into all offline viewer folders.

Deep Scan (Website Mode)

_deep_scan_and_download_assets() runs after the initial website download. It uses BFS convergence:

  1. Scan all text files (JS/CSS/HTML/JSON/SVG) for asset URL references
  2. Filter against pre-built disk index (O(1) set lookup)
  3. Single-pass streamed GET with connection pooling (merged HEAD+GET)
  4. Save new files, re-scan for newly discovered references
  5. Repeat until no new assets found (max 6 rounds)

Optimized with: pre-built disk file index, requests.Session with connection pooling (16 connections, 32 max pool), merged HEAD+GET into single streamed request, up to 20 parallel workers.

Double-Download Prevention

process_images() returns a set of resolved URLs (_resolved_urls). This is passed to _deep_scan_and_download_assets(skip_urls=...) to prevent re-downloading the same assets. Additionally, directory structure is preserved during image saving (no path flattening) so deep scan's path checks match.

Local Preview Server

After downloading in website mode, the app can start a local HTTP server to preview the CYOA in a browser.

  • CORS support with OPTIONS preflight handler

Network & Proxy

Session Management

Shared requests.Session with connection pooling, lazy-initialized per download. Separate session for Cloudflare bypass (via cloudscraper).

Domain Throttling

Per-domain rate limiting with exponential backoff. On failure (429/5xx), backoff doubles (max 30s). On success, backoff halves. Minimum 100ms between requests to the same domain.

Proxy Support

HTTP/HTTPS/SOCKS5 proxy configurable via settings or CLI. Applied to all requests including HEAD checks and asset downloads.

Custom DNS

Patches socket.getaddrinfo globally to resolve through a specified DNS server. Useful for bypassing DNS-level blocks.

CDN-Specific Headers

get_headers_for_url() returns domain-specific headers for 15+ CDN hosts (imgur, discord, cloudinary, catbox, etc.) to reduce hotlink and CDN rejection issues.

HTTP/2 via httpx

v7.3.9 adds optional HTTP/2 for the deep-scan asset downloader. Enable it with --http2 or the GUI HTTP/2 checkbox. The feature uses httpx.Client(http2=True) when httpx[h2] is installed. If the package is missing, the downloader logs a warning and falls back to the existing requests pipeline.

pip install "httpx[h2]"

HTTP/2 is most useful for CDN-heavy sites with many small JS, CSS, font, or image assets. Proxy mode currently falls back to requests for predictable behavior.

BebasDNS Analysis

v7.3.9 removes the previous hosts-file/BebasID resolver. The downloader no longer accepts --hosts-file, no longer shows a Hosts button in the GUI, and no longer parses third-party hosts lists.

BebasDNS is different from BebasID hosts. BebasDNS is a public DNS resolver with DNS-over-HTTPS and DNS-over-TLS variants: default, security, unfiltered, and family. The project documentation states that the default profile blocks malware, trackers, and ads; security focuses on malware and trackers; unfiltered avoids filtering; and family applies stronger filtering for adult, gambling, malware, ads, and other harmful categories.

v7.3.9 includes process-local DNS-over-HTTPS support for BebasDNS presets. The app does not change system DNS, browser DNS, router DNS, hosts files, or firewall rules. Plain DNS can still be configured with --dns 1.1.1.1, while BebasDNS is used through DoH presets such as --bebasdns default.

Recommended future design: add a separate --doh-url option and a GUI DoH dropdown only after implementing RFC 8484 DNS-over-HTTPS resolution with certificate validation, fallback handling, and clear privacy disclosure.

Cache System

Disk-based image cache at ~/.cyoa_downloader/cache/. Keyed by URL hash. Configurable max age (default: 7 days). Stats available via Cache Manager panel in GUI.

  • _cache_get(url) — returns cached bytes or None
  • _cache_put(url, data) — stores bytes with timestamp
  • _clear_image_cache() — wipes all cached files

Download History

Download history stored at ~/.cyoa_downloader/history.json. Records URL, filename, mode, success/failure, timestamp. Used for:

  • Duplicate detection (warn when re-downloading)
  • Batch update checker (HEAD-check previously downloaded URLs for changes)

Reports and Failure Logs

v7.3.9 removes the separate broken_assets_report.html output. Failed asset details now go into the main reporting flow:

  • backup_report.txt receives failed website, cyoap, and deep-scan asset details when it exists.
  • failed_assets.txt is used for non-website outputs when no backup report exists.
  • failed_images.txt remains available for image retry workflows.
  • Old broken_assets_report.html files from previous versions are removed automatically when the output folder is touched by a new run.

This keeps the output folder cleaner and avoids two report files describing the same failure set.

The current implementation updates high-impact GUI microcopy without rebuilding the full UI: download button, output label, browse button, import button, queue warning, and status text. More labels can be added through _tr() and _apply_language().

gallery-dl Design and Current Policy

v7.3.9 keeps gallery-dl optional and disabled by default. The downloader now exposes --gallery-dl off|smart|force, --gallery-dl-path, and --gallery-dl-config. This is intentional. gallery-dl works best with gallery, post, artwork, status, or collection URLs. It is not a universal replacement for a failed direct CDN image URL.

ModeBehaviorRecommended use
offNever call gallery-dl.Default. Best for ordinary ICC/Neocities/Netlify projects.
smartUse only for likely page/post/gallery URLs. Skip obvious raw image CDN URLs.Use when project data contains Pixiv, booru, DeviantArt, X/Twitter, or gallery page links.
forceAllow any HTTP(S) URL to be passed to gallery-dl.Advanced/manual debugging only.

The integration uses --destination, not the invalid --dest alias, and inherits the program proxy through gallery-dl --proxy when a proxy is configured. It still cannot inherit the Python process-local BebasDNS resolver because gallery-dl runs as a separate subprocess.

# Disable gallery-dl, default
python cyoa_downloader_v7_3_7_comprehensive_bugfix.py -u "https://example.com/cyoa/" --gallery-dl off

# Smart mode for page/post/gallery URLs only
python cyoa_downloader_v7_3_7_comprehensive_bugfix.py -u "https://example.com/cyoa/" --gallery-dl smart

# Advanced mode with custom config
python cyoa_downloader_v7_3_7_comprehensive_bugfix.py -u "https://example.com/cyoa/" --gallery-dl force --gallery-dl-config ./gallery-dl.conf

  4. Open http://localhost:8191 in a browser to confirm the service is running.
  5. Use http://localhost:8191/v1 in the CYOA Downloader Cloudflare panel.

Design rules

  • FlareSolverr is not used for every image, CSS, audio, or font request.
  • FlareSolverr solves the page, then cookies and user-agent are copied into the shared requests sessions.
  • reuse-domain keeps one browser session per domain, reducing repeated challenges.
  • temporary creates temporary solves only. It is cleaner but slower.
  • Clear Sessions destroys sessions created by this app instance.
  • Proxy inheritance is optional. Use inherit only when the target site must be reached through the same proxy.

Troubleshooting

HTTP/2 toggle does nothing

Install the optional dependency with:

pip install "httpx[h2]"

Then restart the app. Without this package, the downloader falls back to the standard requests pipeline and logs a warning.

Common Issues

"Could not resolve project data"

The script tried all 5 phases and couldn't find the project JSON. Try:

  • Open the CYOA in a browser, check DevTools Network tab for project.json or large JSON files
  • Try website mode instead (it downloads everything, then you can manually locate the data)
  • Enable AI Assist — Claude can sometimes find non-standard project locations

Cloudflare challenge page

Enable CF Bypass in settings. Requires cloudscraper package. If that fails, try using a browser cookie session (_make_cookie_session).

Images missing in offline playback

Common causes:

  • Single-origin CYOAs that load images from a different CDN — deep scan should catch these
  • Dynamically-loaded images via JavaScript — enable AI Assist for better detection
  • CORS-blocked resources in file:// — use the local preview server instead

Large output files (100+ MB embedded JSON)

Use ZIP mode instead. The embedded JSON base64-encodes all images, inflating size by ~33%. ZIP mode keeps images as separate files.

YouTube audio not downloading

Ensure yt-dlp is installed and the checkbox is enabled. YouTube audio download requires ffmpeg for MP3 conversion. The script auto-detects ffmpeg location.

Viewer not injected

Check that viewer ZIPs are in ~/.cyoa_downloader/offline_viewers/ and the viewers.json registry is intact. Re-run the app to trigger auto-registration.

gallery-dl integration

gallery-dl is optional and defaults to off. It is not a magic fix for every direct CDN image. It works best with post, gallery, or page URLs from supported sites.

ModeBehavior
offNever call gallery-dl.
smartUse gallery-dl only for likely post/gallery/page URLs.
forceTry gallery-dl for more cases. Advanced and slower.

Use --gallery-dl-path when the binary is not on PATH. Use --gallery-dl-config for cookies, extractor options, or site-specific settings.

Local preview server and stale CYOA prevention

The Serve feature is designed for generated website folders. It is not meant to preview a deleted folder from website ZIP mode.

  1. Use Website Folder or --website-folder.
  2. Click Serve or pass --serve.
  3. The server binds to 127.0.0.1 and uses a fresh port when possible.
  4. The browser opens /__clear_cache__ first.
  5. The clear route removes localStorage, sessionStorage, Cache Storage, and service workers for that preview origin.
  6. The browser is redirected to /?cb=timestamp.

Maintenance and known limitations

  • Legacy cwd: run_download() still uses os.chdir(). A process-wide lock prevents concurrent path corruption, but a future refactor should move all output to absolute paths.
  • Cloudflare coverage: Most HTTP paths use the central wrapper, but external tools such as gallery-dl and browser-based FlareSolverr have their own network behavior.
  • AI accuracy: AI suggestions are treated as candidates. The downloader still validates and fetches them normally.
  • Plain API keys: Plain storage is retained only as a fallback. Prefer session, environment variable, or keyring.
  • Blocked networks: BebasDNS can help DNS-level blocks. It cannot guarantee bypass for DPI or Cloudflare challenges.
