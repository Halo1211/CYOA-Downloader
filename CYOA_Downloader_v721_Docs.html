1. Overview & Arsitektur

CYOA Downloader adalah tool desktop untuk download dan packaging Interactive CYOA (Choose Your Own Adventure) dari berbagai hosting. Tool ini mendukung viewer ICC Plus, ICC Remix, cyoap_vue, dan custom React/Vue viewers. Output berupa folder offline yang bisa dibuka langsung di browser tanpa internet.

5. 5-Phase Detection Pipeline

Fungsi get_project_source() mencoba 5 strategi berurutan untuk menemukan data project.json:

Phase 1: Filename Candidates → Phase 2: Regex in HTML → Phase 3: Content Probe → Phase 4: Iframe Recursion → Phase 5: AI Detection

Phase 1 — Filename Candidates (60+ URLs)

build_default_project_candidates() generates candidate URLs berdasarkan URL patterns. Termasuk project.json, project.txt, data.json, subpath variations, dan archive.org patterns. Semua di-HEAD check secara parallel.

Phase 2 — Regex Extraction

Scan HTML source untuk fetch("..."), src="...", dan script tags yang load project data. Match patterns seperti fetch("./project.json") atau <script src="app.js">.

Phase 3 — Content Analysis

Download JS bundles dan scan untuk embedded project data. Gunakan extract_embedded_project_from_js() yang mendeteksi balanced-brace JSON objects di dalam minified JS.

Phase 4 — Iframe Recursion

Cari <iframe> tags di HTML dan recursively apply Phase 1-3 ke iframe URLs. Max depth: 4 levels. Handles cyoa.cafe wrapper URLs.

Phase 5 — AI-Assisted Detection

Jika Phase 1-4 gagal DAN AI Assist aktif: kirim HTML (truncated 8KB) ke Claude API. AI menganalisa patterns (fetch, XHR, script tags) dan return candidate URL. Candidate di-verify via GET request.

6. Image Pipeline A–F

process_images() mendownload semua image + audio yang direferensikan di project.json. Ada 6 layer fallback:

LayerMetodeKapan dipakai
ADirect GETDefault — request langsung dengan domain-specific headers
BDisk CacheURL sudah pernah di-download sebelumnya
CCDN Headers (25+ domain)Domain butuh header khusus (Imgur, Imgchest, Catbox, dll)
DExponential BackoffRate-limited (429) — retry 3× dengan delay 2s→4s→8s
ECookie SessionSite butuh auth cookies (browser-cookie3)
Fgallery-dlImgur, Pixiv, DeviantArt, dll — specialized downloaders

CDN Headers (25+ domains)

get_headers_for_url() returns domain-specific headers untuk bypass hotlink protection. Termasuk: Imgur, Imgchest, Catbox, Imgbb, PostImages, i.redd.it, Cloudinary, Discord CDN, Google Drive, Dropbox, dan lainnya.

Image Deduplication

_check_image_dedup() — SHA256 hash check untuk skip identical files yang sudah didownload di session ini.

7. Audio & YouTube

Audio Fields (26 field names)

_deep_scan_project_assets() walks seluruh JSON tree dan extract audio dari 26+ field names: audio, audioSrc, backgroundMusic, sfx, bgm, voiceover, narration, soundEffect, clickSound, dll.

YouTube Audio (yt-dlp)

Alur: detect bgmId + useAudioURL siblings → classify as YouTube ID or direct URL → download via yt-dlp → convert ke MP3 (ffmpeg) → patch project JSON references.

ffmpeg 5-Layer Detection

_find_ffmpeg() mencari ffmpeg di: (1) PATH, (2) bundled binary, (3) yt-dlp's bundled ffmpeg, (4) common install dirs (win32), (5) pyffmpeg package.

YouTube Stub (Offline)

Untuk offline viewer: inject YouTube IFrame API stub yang fire onYouTubeIframeAPI dan onYouTubeIframeAPIReady callbacks tanpa network access.

9. Offline Viewer Injection

_apply_offline_viewer() — 3-strategy system untuk inject project data ke offline viewer:

Strategy A — Template Markers (ICC Remix)

ICC Remix punya {{ICC_PROJECT_DATA_SCRIPT}} placeholder di index.html. Replace dengan <script> tag berisi project JSON.

Strategy B — Balanced-Brace Injection (ICC Plus)

ICC Plus app.js punya komentar /*! Delete and replace this part...*/ diikuti default state object. Inject project JSON menggantikan default state. Regex: _ICC_MARKER_RE (module-level).

Strategy C — Fetch Interceptor (Fallback)

Inject script ke <head> yang intercept fetch() dan XMLHttpRequest. Ketika viewer request project.json, interceptor return injected data. Works untuk semua viewer types.

Viewer Registry

Viewers disimpan di ~/.cyoa_downloader/offline_viewers/ dengan manifest viewers.json. Auto-register bundled viewers saat startup. Scoring system memilih viewer terbaik berdasarkan: version match (+10), type match (+8), LocalViewer bonus (+5), priority tier (+3).

13. Network Layer

Proxy

Konfigurasi via GUI input atau --proxy flag. Format: http://127.0.0.1:7890 atau socks5://.... Otomatis skip proxy untuk localhost/bare IPs. Rebuild shared session saat proxy berubah.

Custom DNS

_set_active_dns() — patch socket.getaddrinfo globally. Resolves via UDP ke DNS server yang ditentukan (e.g., 1.1.1.1, 8.8.8.8). Berguna untuk bypass ISP blocking.

CF Bypass (Cloudflare)

Toggle di row2. Menggunakan cloudscraper library yang auto-solve JS challenges. Shared session terpisah (_shared_session_cf) dari regular session.

Bandwidth Throttle

_throttle_bandwidth() — sliding window rate limiter. Set via GUI input (KB/s) atau --bandwidth flag. 0 = unlimited. Juga serves sebagai hook point untuk speed graph tracking.

Domain Rate Limiter

_domain_throttle() — per-domain minimum interval (200ms default). Exponential backoff on 429/5xx. Auto-halve backoff on success. Prevents slamming individual servers.

Retry Session

create_retry_session() — requests.Session dengan retry strategy: 3 retries, backoff factor 0.5, retry on 429/500/502/503/504.

Threading Model

GUI thread (tkinter mainloop) + worker thread (download). Worker spawns ThreadPoolExecutor untuk parallel image/asset downloads. Communication via root.after() callbacks. Pause via threading.Event.

Session Management

_get_shared_session(use_cf=False) — lazy-creates dan caches 2 session types: regular (_shared_session) dan cloudscraper (_shared_session_cf). Reset to None saat proxy/DNS berubah → re-created on next use.
